The Logic of Hebbian Learning 
======
<p style="text-align:left;">
  <a href="https://journals.flvc.org/FLAIRS/article/view/130735"> [Paper] </a>
  <a href="https://github.com/ais-climber/neural-semantics"> [Github] </a>
  <a href="https://ais-climber.github.io/files/hebbian_slides.pdf"> [Slides] </a>
  <a href="https://ais-climber.github.io/files/hebbian_poster.pdf"> [Poster] </a>
    <span style="float:right;">
        Knowledge Representation, Dynamic Logic, Neural Networks
    </span>
</p>

**Abstract:** We present the logic of Hebbian learning, a dynamic logic whose semantics are expressed in terms of a layered neural network learning via Hebb’s associative learning rule. Its language consists of modality Tφ (read “typically φ,” formalized as forward propagation), conditionals φ ⇒ ψ (read “typically φ are ψ”), as well as dynamic modalities [φ +]ψ (read “evaluate ψ after performing Hebbian update on φ”). We give axioms and inference rules that are sound with respect to the neural semantics; these axioms characterize Hebbian learning and its interaction with propagation. The upshot is that this logic describes a neuro-symbolic agent that both learns from experience and also reasons about what it has learned.

Knowledge Audit of CDH Network Models
======
<p style="text-align:left;">
  <a href="https://ais-climber.github.io/files/Kisby_CDH_Nets_CBR_Adaptation.pdf"> [Paper Draft] </a>
  <a href="https://colab.research.google.com/drive/1GO2YhEuXCgJ80IhCIr0f5QMCyEDXjTqS"> [Colab] </a>
    <span style="float:right;">
        Python, Tensorflow, CBR Adaptation
    </span>
</p>

**Abstract:** Crafting quality CBR adaptation rules can be challenging. Recent work proposes leveraging neural networks to learn adaptation rules, using the case difference heuristic. But this presents an issue: Are these network models performing CBR adaptation per se, or are they learning different strategies?  In this paper, we formalize falsifiable and testable criteria that differentiate adaptation rules from the alternatives. We demonstrate the use of these criteria via a knowledge audit of various CDH network models.

Neural Networks with CBR Adaptation
======
<p style="text-align:left;">
  <a href="https://ais-climber.github.io/files/Gates_Kisby_NN_with_CBR_Adapt.pdf"> [Report] </a>
  <a href=""> [Github] </a>
    <span style="float:right;">
        Python, Scikit-Learn, CBR
    </span>
</p>

**Abstract:** In this paper, we consider how case-based reasoning can be viewed as a model for integrating domain reasoning in hybrid systems. In particular, case-based reasoning performs domain reasoning (adaptation) on an existing prediction of a different case (retrieval). We suggest a similar architecture for performing adaptation on an existing neural network prediction of a nearby case. We implement CHRISTINE, a system that employs adaptation on neural network predicted ghost cases to predict the price of used cars. Our results about this novel architecture leave the reader wanting, so we consider several things we should improve before this work is conference-ready. On the other hand, our adaptation significantly improves performance using only three simple rules.

Logic of Sizes with Union and Intersection
======
<p style="text-align:left;">
  <a href="https://ais-climber.github.io/files/Kisby_Sizes_Union.pdf"> [Paper] </a>
  <a href="https://github.com/ais-climber/syllogistic-sizes"> [Github] </a>
  <a href="https://ais-climber.github.io/files/Sizes_Talk.pdf"> [Slides] </a>
  <a href="https://ais-climber.github.io/files/Sizes_Poster.pdf"> [Poster] </a>
    <span style="float:right;">
        Syllogistic Logic, Reasoning about Cardinality, Python
    </span>
</p>

**Abstract:** This paper presents the most basic logics for reasoning about the sizes of sets that admit either the union of terms or the intersection of terms. That is, our logics handle assertions All x y and AtLeast x y, where x and y are built up from basic terms by either unions or intersections. We present a sound, complete, and polynomial-time decidable proof system for these logics. An immediate consequence of our work is the completeness of the logic additionally permitting More x y. The logics considered here may be viewed as efficient fragments of two logics which appear in the literature: Boolean Algebra with Presburger Arithmetic and the Logic of Comparative Cardinality.

Confidence in Neural Networks (using CBR)
======
<p style="text-align:left;">
  <a href="https://ais-climber.github.io/files/Gates_CBR_Confidence.pdf"> [Paper] </a>
  <a href=""> [Github] </a>
  <a href=""> [Slides] </a>
    <span style="float:right;">
        Python, Scikit-Learn, CBR
    </span>
</p>

**Abstract:** Determining when to trust black box systems is a well-known challenge. An important factor affecting users’ trust is confidence in system solutions. Previous case-based reasoning (CBR) research has developed criteria for assigning confidence to the solutions of a CBR system. This paper investigates whether such analysis, coupled with factors such as CBR system competence, can be used to predict confidence in the outputs of a black box system, when the black box and CBR systems are provided with the same training data. The paper presents initial strategies for using CBR confidence to predict black box system confidence. An evaluation explores the ability of the strategies to provide useful information and suggests future questions.

Sense-Able
======
<p style="text-align:left;">
  <a href="https://sccapstone.github.io/Sense-Able/"> [Website] </a>
  <a href="https://github.com/ais-climber/sense-able"> [Github] </a>

Sense-Able was a research demo/proof-of-concept app that used a LeddarTech® LIDAR sensor to help those with visual impairments detect objects in front of them.  This was my senior project, in collaboration with Austin Scampini, Jonathan Senn, Kathryn Vincent, and Jinyu Gao, and in service of clients Peter Mumola and Joel Arenson through an industry collaboration with UofSC.  The final product was a program that can detect walls and wall corners by fitting the points to a polynomial curve.  We developed an app for using the sensor, and I personally wrote a more accessible tutorial for using the LeddarTech® SDK.

Nonfinitely-Based Finite Algebras
======
<p style="text-align:left;">
  <a href=""> [Website] </a>
  <a href=""> [Github] </a>
  <a href=""> [Poster] </a>
    <span style="float:right;">
        Equational Logic, Automata
    </span>
</p>

An undergraduate research project I did under the supervision of [George McNulty](https://people.math.sc.edu/mcnulty/).  The goal was to try characterizing finite algebras that can't be axiomatized using equational reasoning (which is, maybe surprisingly, undecidable).  This mostly involved trying to generalize a proof that was common among three graph and automaton algebras in the literature.  The work was funded by the [Magellan Scholar Undergraduate Research Grant](https://sc.edu/about/offices_and_divisions/undergraduate_research/apply_for_funding/our_funding/magellan-scholar-award/index.php).  Unfortunately, I never finished the project, but I did present a poster of my progress at [Discover UofSC](https://sc.edu/about/signature_events/discover_uofsc/).
